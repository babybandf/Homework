# 解题任务执行规范

> ⚠️ 本文档为**实现细节参考**。硬性约束以 [AGENT.MD](AGENT.MD) 为准，本文档不得覆盖 AGENT.MD 的 MUST 条款。

## ⚠️ 本文档的 MUST 条款（其他为参考）

- **M1** 几何绘图 MUST 使用 Python matplotlib 输出 PNG（详见 §3）。
- **M2** Markdown 公式 MUST 使用纯文本格式（详见 §4），MUST NOT 使用 `$...$` / `$$...$$`。
- **M3** 文件命名 MUST 严格遵循附表（见文末）。
- **M4** MD 与 HTML MUST 内容同步（§8）。
- **M5** 任务完成前 MUST 通过 [checker.md](checker.md) 的全部清单并贴出自检报告。

---

## 1. 解析题目
- 仔细阅读题目图片，提取所有已知条件和求解目标
- 确认题目适用的知识范围和难度等级
- 标注关键几何图形、数值和关系

## 2. 生成解题思路
- 分析题目考查的核心知识点
- 设计从已知到未知的推理路径
- 确定关键突破口和辅助线/辅助构造
- 规划分步骤的解题流程

## 3. 生成几何图形（如需要）

> 详细绘图规范见 [geometry_drawing_guide.md](geometry_drawing_guide.md)

- 默认使用 Python matplotlib 绘制静态PNG图
- 备选方案（需用户明确指定）：JSXGraph（交互式）、Manim（动画）
- 图片命名规范：`step{序号}_{描述}.png`
- 插图文字一律使用英文，避免中文字体问题
- 坐标范围必须先计算所有关键点再设置，留10-20%边距

## 4. 编写Markdown解题文档
- 首先展示完整题目
- 按步骤展示解题过程，每个步骤之间用空行隔开
- 适当插入生成的图片
- 使用生动的语言启发思考
- 最后给出完整答案
- **数学公式格式规范**：
  - 使用纯文本表示公式，确保在各种Markdown预览器中兼容显示
  - 分数使用 `(a/b)` 或 `a/b` 格式，如 `(1/2)`、`2/13`
  - 面积比使用 `S△AMN / S△ABC` 格式
  - 平方使用 `²` 上标，如 `(1/2)²`
  - 重要公式使用 `**加粗**` 或 `>` 引用块突出显示
  - 避免使用LaTeX语法（如 `$...$` 或 `$$...$$`），除非确定目标平台支持

## 5. 生成交互式网页

> 详细网页生成规范见 [webpage_guide.md](webpage_guide.md)

- 基于Markdown内容生成HTML网页
- 技术栈：TailwindCSS + JSXGraph 1.8.0 + MathJax 3（均CDN引入）
- 步骤卡片式布局，逐步展示，`display: none` 切换
- 每步配JSXGraph交互几何图 + 2-3个可展开提示问答
- 顶部中英双语切换（`data-i18n` + 翻译字典）
- 支持键盘导航、进度指示器

## 6. 添加解题思路梳理页面
- 在网页最后添加总结页面
- 包含：完整解题思路流程图、解题技巧总结、涉及知识点归纳

## 7. 文件组织规范
- 创建与题目图片同名的文件夹（如题目为`001.png`，则创建`001/`文件夹）
- 所有相关文件存放于该文件夹内：
  - 原题图片
  - 生成的几何图形
  - HTML网页文件
  - Python绘图脚本
  - Markdown解题文档

## 8. 内容同步维护
- **时刻保持HTML与Markdown内容同步**：任何对解题内容的修改必须同时更新到两个文件
- 修改流程：先修改Markdown文档 → 再同步更新HTML网页
- 同步检查点：题目描述、解题步骤、数学公式、配图引用、最终答案
- 建议：在任务完成前进行最终一致性检查

## 9. 测试验证
- 启动本地HTTP服务器测试网页
- 验证所有图片正常显示
- 验证交互功能（步骤切换、提示展开）
- 验证数学公式正确渲染
- 验证响应式布局
- **验证HTML与Markdown内容一致性**

## 附：文件命名规范

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 原题图片 | `{编号}.png` | `001.png` |
| 网页文件 | `{编号}.html` | `001.html` |
| 步骤图片 | `step{序号}_{描述}.png` | `step1_basic.png` |
| Python脚本 | `geometry_figures.py` | `geometry_figures.py` |
| Markdown文档 | `{编号}_solution.md` | `001_solution.md` |
| 文件夹 | `{编号}/` | `001/` |
